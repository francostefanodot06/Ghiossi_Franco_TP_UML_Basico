# 📊 Trabajo Práctico: Modelado UML y Resolución en Java (POO)

Este repositorio contiene la resolución de la guía completa de **Modelado UML y Programación Orientada a Objetos** correspondiente a la **Tecnicatura Universitaria en Programación (UTN)** 🇦🇷.

El objetivo del proyecto es llevar requerimientos de lógica de negocio y diagramas de clases a código real en **Java**, aplicando de forma estricta los pilares de la POO y controlando las interacciones y el ciclo de vida de los objetos.

---

## 🛠️ Tecnologías y Herramientas
* **Lenguaje:** Java (JDK 8 o superior).
* **Entorno de Desarrollo (IDE):** Apache NetBeans  (sobre entorno Linux).
* **Diseño:** Diagramas de Clases en UML 2.0.
* ## 📂 Estructura del Proyecto y Ejercicios

El código está estructurado en paquetes independientes, donde cada uno resuelve una problemática específica de modelado y asociación:

* **📦 Ejercicio 1:** Relación de documentación entre `Titular`, `Pasaporte` y `Foto`.
* **📦 Ejercicio 2:** Modelado de hardware/electrónica con `Celular`, `Bateria` y `Usuario`.
* **📦 Ejercicio 3:** Sistema editorial gestionando relaciones entre `Libro`, `Autor` y `Editorial`.
* **📦 Ejercicio 4:** Simulación financiera con las entidades `Banco`, `Cliente` y `TarjetaDeCredito`.
* **📦 Ejercicio 5:** Relación de componentes lógicos de hardware: `Computadora`, `PlacaMadre` y `Propietario`.
* **📦 Ejercicio 6:** Sistema de reservas gastronómicas interactuando entre `Cliente`, `Mesa` y `Reserva`.
* **📦 Ejercicio 7:** Control automotriz (encendido de `Motor`, gestión del `Vehiculo` y asignación de `Conductor`).
* **📦 Ejercicio 8:** Simulación de seguridad digital mediante `Documento`, `FirmaDigital` y `Usuario`.
* **📦 Ejercicio 9:** Gestión de turnos médicos interactuando entre `Paciente`, `Profesional` y `CitaMedica`.
* **📦 Ejercicio 10:** Módulo bancario con validación de seguridad entre `CuentaBancaria`, `ClaveSeguridad` y `Titular`.
* **📦 Ejercicio 11:** Modelado multimedia de un `Reproductor` gestionando listas de `Cancion` y colecciones de un `Artista`.
* **📦 Ejercicio 12:** Módulo tributario implementando una `Calculadora` para determinar el `Impuesto` de un `Contribuyente`.
* **📦 Ejercicio 13:** Sistema de accesos mediante generación y validación de un `CodigoQR` para un `Usuario`.
* **📦 Ejercicio 14:** Flujo de trabajo audiovisual compuesto por un `EditorVideo`, la gestión de un `Proyecto` y el proceso de `Render`.

---

## 📐 Conceptos de POO Aplicados
* **Encapsulamiento:** Uso estricto de modificadores de acceso (`private`), constructores parametrizados y métodos de acceso público (*getters* y *setters*).
* **Relaciones de Objetos:** Implementación práctica en código de dependencias fuertes (**Composición**, donde el ciclo de vida depende del objeto padre) y dependencias débiles (**Agregación**).
* **Modularidad:** Separación de responsabilidades limpia dividida por paquetes lógicos.
